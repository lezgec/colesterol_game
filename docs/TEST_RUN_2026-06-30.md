## Validaciones de seguridad aplicadas

- Los detalles de errores internos quedan ocultos salvo cuando `APP_DEBUG=true`.
- `require_csrf_token()` valida por header `X-CSRF-Token` o campo POST; ya no lee el cuerpo JSON.
- Reportes, rankings, historial, dashboards y perfil escapan datos dinámicos antes de insertarlos con `innerHTML`.
- Avatares aceptan solo JPG, PNG y WebP.
- Avatares se re-codifican con GD antes de guardarse.
- Al subir un avatar nuevo se elimina el avatar personalizado anterior si pertenece al directorio permitido.
